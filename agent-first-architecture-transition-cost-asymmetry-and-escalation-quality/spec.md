# 에스컬레이션 품질 시뮬레이터: Workflow vs Agent

> 원본 아이디어: Agent-First 아키텍처 전략 - 전환 비용의 비대칭성과 에스컬레이션 품질

## 검증 목표
Agent의 confidence 기반 에스컬레이션(5가지 판단 순간)이 Workflow의 규칙 기반 에스컬레이션보다 품질이 높다는 가설을 시뮬레이션으로 검증한다.

## 범위
- 포함:
  - 고객 지원 시나리오 세트 (단순/복합/애매한 케이스)
  - Workflow 엔진: 하드코딩된 규칙 기반 분기 + 에스컬레이션
  - Agent 엔진: 5가지 판단 순간(Intent 해석, Tool 선택, Context 충분성, 결과 검증, 사람 개입 판단)에서 confidence 기반 에스컬레이션
  - 결과 비교 대시보드 (정확도, 불필요 에스컬레이션, 놓친 에스컬레이션)
  - CLI로 시뮬레이션 실행 + 웹 대시보드로 결과 시각화
- 제외:
  - 실제 LLM 호출 (confidence 점수는 시뮬레이션)
  - 멀티 에이전트 시스템
  - 실제 고객 데이터 연동
  - AX 플랫폼 통합

## 기술 스택
- Python 3.12+ (`uv`로 관리) — 시뮬레이션 엔진
- 단일 HTML + vanilla JS — 결과 시각화 대시보드
- JSON — 시나리오/결과 데이터 저장
- 이유: 시뮬레이션 로직 중심이므로 무거운 프레임워크 불필요. Python 표준 라이브러리만으로 핵심 구현 가능.

## 완료 기준
- [x] 최소 10개 시나리오(단순 5 / 복합 3 / 엣지 2)가 두 엔진에서 처리됨
- [x] Agent 엔진이 5가지 판단 순간 각각에서 confidence 점수를 산출하고 에스컬레이션 결정을 내림
- [x] Workflow 엔진 대비 Agent 엔진의 에스컬레이션 품질 차이가 수치로 비교됨 (정확도, 불필요 에스컬레이션 비율, 놓친 에스컬레이션 비율)
- [x] 웹 대시보드에서 시나리오별 처리 과정과 비교 결과를 시각적으로 확인 가능
- [x] CLI에서 `uv run main.py`로 시뮬레이션 실행 후 결과 확인 가능
