### 3-2. 스스로 평가
1. **"오 되네" vs "뭐야 이게"?** → API 레벨에서는 "오 되네". 단, 브라우저에서 실제 사용성은 직접 확인 필요. UI 스타일링은 깔끔하게 구현했으나 시각적 검증 미완.
2. **검증 목표가 실제로 검증되는가?** → 예. markdown-it 소스 라인 매핑 → contenteditable 편집 → .md 동기화 경로가 API 테스트로 확인됨. 드래그→인라인 스타일 반영도 확인.
3. **출력/UI가 깔끔한가?** → HTML 구조상 다크 테마 + 슬라이드 프리뷰 + 소스 에디터 + diff 패널 구성. 브라우저 확인 불가하여 시각적 품질 확신 부족.
4. **빠진 게 있는가?** → 드래그 핸들이 CSS의 left:-24px이라 좁은 화면에서 잘릴 수 있음. contenteditable 편집 시 블록 내부의 bold/italic 마크다운 서식이 날아갈 수 있음(인라인 레벨 매핑은 스코프 외).
